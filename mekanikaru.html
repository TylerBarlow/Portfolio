Summary

The idea which eventually became Mekanikaru started as a desire to create a cooperative, interactive experience that put one user in charge of relaying instructions, and another in charge of physical inputs. The result was a robot fight theamed game, with one user viewing a screen and the other interfacing with a control panel.

Constraints

The project was required to make use of: an Arduino, an interface with processing on a computer and at least 3 different sensors. Interaction needed to consist of: input, output, and a GUI. The goal was to create an interactive system.

Inspiration

We set out to design an interactive game that was sufficiently complex enough to warrant using 4 Arduino micro-controllers. We took inspiration from the video game "Keep talking and no one explodes". Our goal was to create a physical variation of the game's virtual interface.

Concept

Our solution was to design and implement 4 different physical "modules" that would make up 4 stages of the game. Each of these modules would be a self contained mini-game, requiring communication with the second player to complete.

Programming

The microcontroller was programmed in Wiring; a Java based language for coding Arduino, and the computer side in Processing. Difficulties arose in controlling a large number of input/output pins, and in controlling 4 Arduinos simultaneously.

The constrained number of pins useable on the Ardiuno board meant some creative soultions were needed. The jack module required 35 different possible detections. A to 1, A to 2... B to 1, B to 2 ect. The solution was to power each cord briefly, and check for input on each of the 7 inputs. This reduced the number of connections needed to 13.

Physical Model

After prototyping was complete and we knew the idea was feasable, the physical model was constructed. All 4 modules were placed on the same surface slanted forward, similar to a machine control panel. This was chosen after we decided on a theam of fighting robots, the control panel look was to match the concept. The box and panels were hand cut out of plywood and MDF.

Keeping with our modular system, each panel was built seperatly and slotted into holes in the main box. This made it possible to wire it up outside of the box, and remove a panel for easy troubleshooting.

Care was taken to ensure each component of each module was well defined, and had a planned location of installation.

This was important to prevent errors and confusion, as roughly 40 different components were to be connected inside the box.
